U6 Functions

U6L1 Intro to Functions

  • Function:
    • Give a "chunk" of code a name. Every time you type in that function name in code, it repeats the chunck.
    • A subroutine consisting of one or more lines of code. The subroutine can be called by typing in the name of the subroutine.
  • General Function Structure:
    • function functionName(){
      code that will run...
      }
    • Another way to write it: let functionName = () => {
      code in here...
      }
    • Calling the function: functionName();
  • You can call a function more than once

U6L2 Parameters and Arguments

  • Parameters are variables that you can "require" when a function is called
  • Arguments are the values you "pass" to parameters when you call the function
  • Example:
    • function calc_area_rect(l, w){
      let area = l*w;
      alert("The area of the rectangle with length" + l + " and width " + w + " is " + area);
      }
    • calc_area_rect(5, 10); //alert will give 50 as answer (5 is length, 10 is width)
  • To see another example, open the console in U6L1Practice.html
  • You can call a function as many times as you want, and each call can have different parameters
  • Parameters can also be strings
    • function greet(name, job){
      alert("Welcome, " + name + "! We are glad to have a " + job + " at the convention today!"
      }
    • greet("Bob", "surgeon"); //message will be "Welcome, Bob! We are glad to have a surgeon at the conventoin today!"
  • If you specify more arguments than there are parameters, you will get an error message
  • Typically, all of the function calls are placed at the end of the document.

U6L3 Return Statements

  • While you can specify a specific way to print the output of a function (ex: an alert), this is "hard-coded" into the code itself--in other words, you have already specified how the information is going to be printed (you would have to change the output type if you wanted it to be console.log() or the response to a button click)
    • In other words, if you fix the calculation code of a function with the displaying of the info, you are kind of stuck--if you want to change the output type, you have to go and rewrite the output (ex: make alert(output code); --> console.log(output code);. Or if you just want a function to do something and just give you an answer back, a return is often more practical
  • Ex of return statement: Math.pow(3,4); //means 3^4 and returns that answer value
  • Syntax for return statements: return variableName;
  • You can manipulate the returned variable outside of the function (see U6L3Practice.html)
  • Returning a value establishes that value as the "final answer" to this function and allows it to be called later in multiple different ways outside of the function
  • Example:
    • function greet(name, job){
      return ("Welcome, " +name+ " the " +job+ "!");
      }
    • alert(greet("Bob", "surgeon"));
  • U6L3 Practice

U6L4 Buttons, getElementById, and Button Functions

  • U6L4 Practice and Assessment
  • You can reference almost anything in the window (the html document) using JS--by ID, element, class, etc.
  • In a JS document (or in the console), typing document will reference the entire html document. document.___ allows you to call a specific item/group by a property (ex: document.getElementById("_elementID_");)
  • If you type document.getElementById("_elementID_"); in the console, you get the html code for the code with that ID
  • To access the selected code w/ getElementById later, you have to store it in a JS variable
  • There is no set convention for naming element IDs. However, major companies usually use all lowercase and hyphens to separate the words in ID and class names (ex: click-me) in HTML and use camelCase (ex: clickMe) in JS.
  • ID and class names should also be meaningful so you and other people can understand your code later
  • If you have buttons or other things that interact with JS after the page load, you need to type defer after including the source code for the script. This tells the JS code to load after the rest of the document is loaded (this is very important if the script code is placed earlier in the code than what the code references).
  • Ex linking JS: <script src="buttons.js" defer> </script >

U6L5 Text Areas

  • U6L5 Practice
  • Technically you should use forms for text inputs, but text areas are faster. You can both get input and give outputs using text boxes.
  • cols="_number_" tells the number of columns to put in the text box (the numer of characters in one row of textbox based on initial size); rows="_number_" gives number of rows (NOTE: this is the syntax for inside the element tags, NOT CSS)
  • You can use .value to get or define (depends on context) the value inside of an element (for instance, the text inside a text box)
